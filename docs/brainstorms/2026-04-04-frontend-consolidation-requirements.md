# Frontend Consolidation: Merge Sophia MVP into `frontend/`

## Problem Frame

The workspace contains two completely separate frontend applications: `frontend/` (DeerFlow agent IDE, unused by Sophia) and `AI-companion-mvp-front/` (the actual Sophia product). They share no code, no auth, no state. The result is a fragmented workspace where:

- `make dev`, Docker, CI, and the Makefile wire to `frontend/` — but the product lives in `AI-companion-mvp-front/`
- Developers must run two separate frontends manually
- The PowerShell scripts (`sophia-dev.ps1`, `start-all.ps1`) use `npm` against `AI-companion-mvp-front/`, while all infrastructure expects `pnpm` against `frontend/`
- Surface boundary guardrails exist solely to keep the two from bleeding into each other — a constraint that only exists because there are two in the first place
- The upstream repo (`davidelaverga/Sophia-Agent`) has a single `frontend/` directory

The target state: one `frontend/` directory containing the Sophia product, wired to all existing infrastructure (Makefile, Docker, CI, nginx), using pnpm and modern framework versions.

## Requirements

**Code Migration**

- R1. All source code from `AI-companion-mvp-front/src/` becomes the contents of `frontend/src/`, replacing the DeerFlow source entirely
- R2. Sophia's `middleware.ts` (locale detection + debug-route blocking) moves to `frontend/middleware.ts`
- R3. Sophia's `tailwind.config.ts`, `postcss.config.js`, and `vitest.config.ts` move to `frontend/`, adapted for the target toolchain
- R4. Sophia's test files (`tests/`, test configs, Playwright config) move to `frontend/tests/`
- R5. All Sophia static assets (`public/`) move to `frontend/public/`, replacing DeerFlow assets

**Framework Upgrade**

- R6. The consolidated `frontend/` uses Next.js 16 + React 19 + TypeScript 5.8 (matching current DeerFlow versions)
- R7. All Sophia components are updated to work with React 19 (breaking changes: `ref` as prop instead of `forwardRef`, `useFormStatus`, async components, etc.)
- R8. Tailwind CSS upgrades to Tailwind 4 (matching DeerFlow). Sophia's `tailwind.config.ts` is replaced with Tailwind 4's CSS-first `@theme` configuration. Utility class syntax differences are resolved during migration.

**Package Manager**

- R9. The consolidated `frontend/` uses pnpm (with `pnpm-lock.yaml`), not npm
- R10. All Sophia dependencies are added to the pnpm-managed `package.json`; DeerFlow-only dependencies (CodeMirror, @xyflow/react, @langchain/*, @radix-ui/*, shiki, rehype-*, remark-*, katex, better-effort-json-parser, gsap, ogl) are removed unless also needed by Sophia
- R11. The `package.json` `name` changes to `sophia-frontend` and `packageManager` field specifies `pnpm@10.26.2`

**Auth Migration (Supabase → Better Auth)**

- R12. All Supabase auth dependencies (`@supabase/auth-helpers-nextjs`, `@supabase/auth-helpers-react`, `@supabase/ssr`, `@supabase/supabase-js`) are removed
- R13. Better Auth is configured as the auth provider with session management, login/signup flows, and OAuth callback handling
- R14. Route protection middleware uses Better Auth session checks instead of Supabase
- R15. All auth-dependent components (login page, auth callback, protected route wrappers, user context hooks) are re-implemented against Better Auth
- R16. Environment variables shift from `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` to `BETTER_AUTH_SECRET` + Better Auth config

**Capacitor Support**

- R17. `capacitor.config.ts` moves to `frontend/capacitor.config.ts`, updated to reflect the new directory structure (`webDir` adjusted to Next.js output path)
- R18. Capacitor dependencies (`@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`, `@capacitor/android`, `@capacitor/haptics`, `@capacitor/splash-screen`, `@capacitor/status-bar`) remain in `package.json`
- R19. `next.config.js` preserves the Capacitor conditional build logic (`output: 'export'` when `CAPACITOR_BUILD=true`, `unoptimized` images, ignored ESLint/TS errors during Capacitor builds)
- R20. `ios/` and `android/` directories remain gitignored (generated by `npx cap sync`)

**Configuration Merge**

- R21. `next.config.js` merges DeerFlow's production-hardened security headers (CSP, HSTS, X-Frame-Options, etc.) with Sophia's Capacitor build logic and `optimizePackageImports`
- R22. `tsconfig.json` uses modern settings (`es2022` target, `Bundler` module resolution) with Sophia's Capacitor exclusions (`ios/`, `android/`, `capacitor-cordova-*`). `strict: true` is deferred — Sophia was written with `strict: false`, enabling it would surface hundreds of type errors and belongs in a separate pass
- R23. ESLint config uses DeerFlow's flat config format (ESLint 9 + `typescript-eslint`) adapted for Sophia's codebase (ignoring `components/ui/**` as DeerFlow does)
- R24. `frontend/.env.example` is created or updated with all required Sophia env vars (Cartesia, Deepgram, Stream, Mem0, Better Auth)

**Infrastructure Wiring**

- R25. Root `Makefile` targets (`install`, `dev`, `start`, `stop`) continue to work unchanged — they already reference `frontend/` and `pnpm`
- R26. `scripts/serve.sh` continues to work unchanged — it already uses `cd frontend && pnpm run dev`
- R27. `scripts/sophia-dev.ps1` and `scripts/start-all.ps1` are updated to reference `frontend/` instead of `AI-companion-mvp-front/` and use `pnpm` instead of `npm`
- R28. `docker/docker-compose-dev.yaml` and `docker/docker-compose.yaml` continue to work unchanged — they already reference `frontend/Dockerfile`
- R29. `frontend/Dockerfile` continues to work unchanged — Sophia uses the same build pattern (`pnpm install` → `pnpm build` → `pnpm start`)

**Guardrail Removal**

- R30. `docs/MVP_FRONTEND_SURFACE_BOUNDARY.md` is deleted (no longer relevant with a single frontend)
- R31. `scripts/check-sophia-surface-boundary.js` is deleted
- R32. `frontend/src/core/sophia/` empty-directory marker is removed (Sophia code now lives in `frontend/` directly)

**CI/CD**

- R33. The E2E workflow currently at `AI-companion-mvp-front/.github/workflows/memory-highlights-e2e.yml` moves to `.github/workflows/` and is updated to run from `frontend/` using `pnpm`
- R34. `.github/copilot-instructions.md` is updated to reflect that `frontend/` is Sophia (remove references to DeerFlow frontend behavior, Better Auth secret requirement stays)

**Cleanup**

- R35. `AI-companion-mvp-front/` directory is deleted after all code is migrated and verified
- R36. `frontend/CLAUDE.md`, `frontend/AGENTS.md`, and `frontend/README.md` are updated to describe Sophia, not DeerFlow
- R37. Root `CLAUDE.md` repository structure section is updated to remove `AI-companion-mvp-front/` references

## Success Criteria

- `make dev` starts the Sophia companion frontend at `localhost:2026` (via nginx proxy from `localhost:3000`)
- `pnpm lint`, `pnpm typecheck`, and `BETTER_AUTH_SECRET=... pnpm build` all pass from `frontend/`
- All existing Sophia routes work: `/`, `/chat`, `/session`, `/recap`, `/onboarding`, `/settings`, `/history`, `/memories`, `/reflections`
- Better Auth login/signup/logout flows work
- Capacitor build succeeds: `CAPACITOR_BUILD=true pnpm build && npx cap sync ios`
- Docker build succeeds: `docker compose -f docker/docker-compose-dev.yaml build frontend`
- Existing Vitest tests pass from `frontend/`
- Playwright E2E tests pass (adapted for new auth)
- `AI-companion-mvp-front/` directory no longer exists
- No code references `AI-companion-mvp-front` anywhere in the repository

## Scope Boundaries

- Backend is not touched — `backend/`, `gateway/`, `langgraph.json` remain unchanged
- `voice/` server is not touched — it communicates with the frontend via HTTP/WebRTC regardless of directory location
- `skills/`, `users/`, `docs/specs/` are not touched
- Root-level Sophia architecture docs (`01_architecture_overview (new).md`, etc.) are NOT reorganized in this work — that's a separate cleanup
- No new features are added — this is a structural migration + framework upgrade + auth migration
- The app should work identically from the user's perspective after the move, despite internal framework version changes

## Key Decisions

- **Sophia code is the base**: The DeerFlow agent IDE frontend is removed; Sophia's source replaces it entirely
- **pnpm (match upstream)**: Aligns with Makefile, Docker, CI, upstream repo, and `.github/copilot-instructions.md`
- **Next.js 16 + React 19**: Modernizes the stack; React 19 has meaningful improvements (ref forwarding, form actions, concurrent features)
- **Better Auth replaces Supabase**: User decision — Better Auth infrastructure already exists in DeerFlow's frontend shell
- **Capacitor preserved**: iOS/Android native builds continue to work from `frontend/`
- **Tailwind 4 upgrade**: Migrate Sophia's Tailwind 3 to 4 during consolidation rather than deferring a second migration

## Dependencies / Assumptions

- React 19 compatibility: Some Sophia components may use patterns deprecated in React 19 (e.g., `forwardRef`, string refs, default props). These need updating during R7.
- Tailwind 3 → 4 gap: If DeerFlow uses Tailwind 4, Sophia's Tailwind 3 utility classes may need syntax updates (Tailwind 4 changed config format and some class names).
- Better Auth: The DeerFlow frontend has Better Auth installed but not activated. The auth migration (R12–R16) is net-new implementation work, not a port. This is the highest-risk scope item.
- Stream.io and other Sophia SDKs must work with React 19. Verify `@stream-io/video-react-sdk` React 19 compatibility.
- `@ai-sdk/react` must work with React 19. Verify Vercel AI SDK compatibility.
- **Simultaneous breaking changes risk**: React 18→19, Next.js 14→16, possibly Tailwind 3→4, and Supabase→Better Auth all at once. Planning should consider whether to phase these (e.g., move code first with pinned versions, then upgrade) vs. do them all in one pass.
